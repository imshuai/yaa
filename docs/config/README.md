## 1. 概述

配置系统是 Yaa! Runtime 的基础设施层，负责定义、加载、校验、合并和热更新所有运行时配置。

### 1.1 设计理念

| 理念 | 说明 |
|------|------|
| **Config over Code** | 优先通过配置扩展系统行为，而非修改代码 |
| **多源合并** | 配置文件 + 环境变量 + 命令行参数，优先级递增 |
| **安全优先** | 敏感信息通过环境变量引用，不硬编码到配置文件 |
| **热更新** | 支持文件监听，配置变更无需重启 Runtime |
| **多格式兼容** | 主推 YAML，同时兼容 TOML 和 JSON |
| **向后兼容** | 新增字段使用默认值，不破坏旧配置 |
| **零 CGO** | 使用纯 Go 库，保证交叉编译和 Windows 兼容 |

### 1.2 核心能力

```text
┌──────────────────────────────────────────────────────┐
│                    Config System                       │
│                                                        │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐            │
│  │  Loader  │→ │ Validator │→ │  Merger  │            │
│  │ (多格式) │  │ (校验)    │  │ (默认值)  │            │
│  └──────────┘  └──────────┘  └────┬─────┘            │
│                                   │                    │
│  ┌──────────┐  ┌──────────┐  ┌───▼──────────────┐   │
│  │ Watcher  │  │ EnvVar   │  │ Effective Config  │   │
│  │ (热更新) │  │ Resolver │  │ (合并后的最终配置)  │   │
│  └──────────┘  └──────────┘  └───────────────────┘   │
└──────────────────────────────────────────────────────┘
```

### 1.3 配置层级与优先级

```text
优先级递增（后者覆盖前者）：
┌──────────────────────────────────────────────────────┐
│  1. 内置默认值 (Default Values)                       │  最低
│  2. 配置文件 (YAML / TOML / JSON)                     │
│  3. 环境变量引用 (${VAR_NAME} 在配置文件中展开)        │
│  4. 命令行参数 (--flag value)                         │  最高
└──────────────────────────────────────────────────────┘
```

**优先级规则：**

| 层级 | 来源 | 覆盖方式 | 示例 |
|------|------|----------|------|
| 1 | 内置默认值 | 代码中定义 | `addr: ":8080"` |
| 2 | 配置文件 | 按字段覆盖 | `addr: ":9090"` |
| 3 | 环境变量引用 | 在配置文件值中展开 | `${YAA_HTTP_ADDR}` → `:9090` |
| 4 | 命令行参数 | 按字段覆盖 | `--runtime.api.http.addr :7070` |

**注意：** 环境变量引用（第 3 层）是在配置文件解析后、校验前展开的。它本质上是配置文件值的一部分，只是值来源是环境变量。命令行参数（第 4 层）在所有配置文件处理完成后覆盖最终结果。
